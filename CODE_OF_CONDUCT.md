# Code of Conduct

This project is maintained by two developers. This document captures how we want to work together.

## Working Together

- Be direct. If something is unclear or seems wrong, say so early.
- Critique the code, not the person writing it.
- If you leave a review comment, explain why, not just what.
- If you are unsure about a decision, open a discussion before merging.
- Acknowledge mistakes without dwelling on them. Fix them and move on.

## Code Reviews

- Reviews exist to improve the code, not to assert authority.
- Approving a PR means you have actually read it and are comfortable with the change.
- If something is a suggestion rather than a blocker, mark it clearly as such.

## Communication

- Keep technical discussions in issues or PR comments where both contributors can reference them later.
- If something is agreed verbally, write it down in the relevant issue or PR.
